# 减肥AI助理 - MVP PRD

## 1. 产品概述

### 1.1 产品定位
一款让用户「轻松减肥、量化减肥」的AI助理应用。

### 1.2 核心价值
- **拍照就知道吃多少** - AI识别食物热量
- **AI陪你减肥** - 大模型驱动的鼓励助手
- **量化看得见** - 热量预算、体重曲线、目标预测

### 1.3 Slogan
**「轻松减肥，AI陪你」**

---

## 2. 用户故事

### 2.1 目标用户
- 有减肥需求的人群（18-45岁）
- 想要轻松记录饮食的人
- 需要监督和鼓励的减肥者

### 2.2 用户场景

| 场景 | 用户行为 | APP响应 |
|------|----------|---------|
| 第一次打开APP | 回答3个问题建立档案 | 自动计算每日预算 |
| 吃早餐 | 拍照 | AI识别食物，显示热量，加入今日记录 |
| 吃午餐 | 拍照或搜索 | AI识别，显示今日剩余预算 |
| 打卡体重 | 记录体重 | 显示体重曲线，AI预测目标时间 |
| 心情低落 | 和AI聊天 | 大模型给予鼓励和建议 |
| 吃多了 | 记录饮食 | AI安慰并给建议 |

## 3. 功能需求

### 3.1 用户档案

**功能描述：** 用户输入基础信息，系统自动计算减脂目标

**数据项：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| nickname | string | 是 | 用户昵称 |
| gender | enum | 是 | 男/女 |
| age | int | 是 | 年龄 |
| height | float | 是 | 身高(cm) |
| current_weight | float | 是 | 当前体重(kg) |
| target_weight | float | 是 | 目标体重(kg) |
| target_date | date | 否 | 目标达成日期（可选） |

**AI计算规则：**
```
1. 计算BMI = 体重(kg) / 身高(m)²
2. 计算BMR（基础代谢率）：
   - 男: BMR = 66 + (13.7 × 体重) + (5 × 身高) - (6.8 × 年龄)
   - 女: BMR = 655 + (9.6 × 体重) + (1.8 × 身高) - (4.7 × 年龄)
3. 计算TDEE（每日总消耗）= BMR × 活动系数(1.2-1.9)
4. 设置热量缺口 = TDEE - 500kcal
5. 计算预计达成时间（按每天-500kcal ≈ 每周减0.5kg）
```

**交互流程：**
1. 一个页面，包含以上所有字段
2. 实时显示计算结果（预估目标时间）
3. 提交后不可修改，仅客服可调整

### 3.2 饮食记录（核心功能）

**功能描述：** 用户通过拍照或搜索记录每日饮食，系统AI识别并计算热量

#### 3.2.1 拍照识别

**流程：**
```
用户点击拍照 → 拍照 → AI识别食物 → 显示识别结果 → 用户确认 → 加入记录
```

**AI识别内容：**
| 输出字段 | 说明 |
|----------|------|
| food_name | 食物名称 |
| calories | 热量(kcal) |
| protein | 蛋白质(g) |
| fat | 脂肪(g) |
| carbs | 碳水(g) |
| confidence | 置信度(0-1) |
| portion_tip | 份量建议（如"约1碗米饭"） |

**识别失败处理：**
- 置信度 < 0.6 时，提示用户手动搜索
- 识别出多种食物时，展示列表供选择

#### 3.2.2 手动搜索

**流程：**
```
用户输入关键词 → 搜索食物库 → 选择食物 → 选择份量 → 加入记录
```

**食物库字段：**
| 字段 | 说明 |
|------|------|
| name | 食物名称 |
| calories | 热量(kcal/100g) |
| protein | 蛋白质(g/100g) |
| fat | 脂肪(g/100g) |
| carbs | 碳水(g/100g) |
| category | 分类（主食/肉蛋/蔬菜/水果/饮品/零食） |

#### 3.2.3 份量选择

**方式：**
- 默认份量单位：个/碗/杯/份（根据食物类型自动匹配）
- 用户可手动输入克数
- 热量自动按比例计算

#### 3.2.4 餐次分类

**分类：** 早餐 / 午餐 / 晚餐 / 加餐

**逻辑：**
- 根据时间自动建议餐次
- 用户可手动调整

### 3.3 热量预算展示

**功能描述：** 实时展示用户今日热量摄入与预算

**展示形式：** 圆环进度图

**数据：**
| 字段 | 说明 |
|------|------|
| daily_budget | 每日目标（kcal） |
| consumed | 已摄入（kcal） |
| remaining | 剩余（kcal） |
| protein_consumed | 已摄入蛋白(g) |
| protein_target | 目标蛋白(g) |
| fat_consumed | 已摄入脂肪(g) |
| carbs_consumed | 已摄入碳水(g) |

**状态设计：**
| 状态 | 显示 |
|------|------|
| < 80% | 绿色圆环，正常 |
| 80-100% | 黄色圆环，接近目标 |
| > 100% | 红色圆环，已超标 |

---

### 3.4 体重记录

**功能描述：** 用户每日记录体重，系统生成曲线并预测目标达成时间

**数据项：**
| 字段 | 类型 | 说明 |
|------|------|------|
| weight | float | 体重(kg) |
| recorded_at | datetime | 记录时间 |

**展示：**
- 体重曲线图（近7天/30天/全部）
- 较昨日变化（+0.2kg / -0.3kg）
- 累计变化（已减 X kg）
- 距离目标（还剩 Y kg）

**AI预测：**
```
预计达成时间 = 距离目标 / 近7日平均减重速度
```

**特殊情况：**
- 体重上升时，提示可能原因（水肿/吃多了/生理期等）
- 连续3天上升，预警并建议调整

### 3.5 AI鼓励助手

**功能描述：** 大模型驱动的智能鼓励系统，像朋友一样陪伴用户减肥

#### 3.5.1 触发机制

| 触发时机 | 触发条件 | 鼓励类型 |
|----------|----------|----------|
| 饮食打卡完成 | 记录完一餐 | 即时鼓励 |
| 体重打卡完成 | 记录体重 | 进度鼓励 |
| 达成小目标 | 减了1kg/坚持7天 | 成就鼓励 |
| 吃多了 | 单餐超标50% | 安慰鼓励 |
| 连续没打卡 | 2天以上未记录 | 唤醒鼓励 |
| 用户主动聊天 | 发送消息 | 智能回复 |
| 每日首次打开 | 打开APP | 早安/午安/晚安 |

#### 3.5.2 大模型Prompt设计

**System Prompt：**
```
你是一个温暖、幽默的减肥鼓励助手，名叫「小减」。

用户正在减肥，你的任务是：
1. 鼓励用户坚持减肥
2. 用轻松、朋友般的方式说话
3. 不要说教、不要太正式
4. 可以适当用emoji
5. 回复要简短（50字以内）
6. 如果用户情绪低落，要给予更多关心

用户信息：
- 名字：{user_name}
- 目标体重：{target_weight}kg
- 当前体重：{current_weight}kg
- 今日已吃：{consumed}kcal / {budget}kcal
- 已坚持：{streak}天

当前场景：{scenario}

请生成一句鼓励的话。
```

#### 3.5.3 记忆系统

**存储内容：**
- 用户对话历史（最近20条）
- 用户重要信息摘要（过敏食物、偏好、说过的话）
- 用户情绪状态标记

**调用时机：**
- 每次大模型对话前，注入记忆上下文
- 鼓励触发前，读取近期状态

**技术方案：**
- 向量数据库（Chroma）存储对话
- 关键信息提取存入PostgreSQL
- MemGPT框架管理记忆

#### 3.5.4 降本策略

| 策略 | 说明 |
|------|------|
| 缓存鼓励语 | 相同场景的鼓励语缓存5分钟 |
| 限流 | 每用户每分钟最多生成2次 |
| 简单场景用模板 | 低复杂度场景用预设模板 |
