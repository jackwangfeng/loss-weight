### 交互说明

| 元素 | 交互 |
|------|------|
| 所有输入框 | 点击弹出键盘/选择器 |
| 性别选择 | 单选按钮，实时切换 |
| 日期选择 | 点击弹出日期选择器 |
| 开始减肥按钮 | 校验必填项 → 提交 → 进入首页 |

### 计算逻辑（实时显示）

```javascript
// 用户输入时实时计算
BMI = weight / (height/100)²

BMR = 
  男：66 + (13.7×weight) + (5×height) - (6.8×age)
  女：655 + (9.6×weight) + (1.8×height) - (4.7×age)

TDEE = BMR × 1.2（默认久坐）

每日预算 = TDEE - 500kcal

预计周数 = (当前体重 - 目标体重) / 0.5
```

## 交互流程总览

### 核心流程

```
首次启动
   ↓
建立档案（1页）
   ↓
首页（2页）
   │
   ├─→ 拍照记录（3页）
   │        ↓
   │     确认添加
   │        ↓
   │     返回首页
   │
   ├─→ 搜索添加（4页）
   │        ↓
   │     确认添加
   │        ↓
   │     返回首页
   │
   ├─→ 体重记录（5页）
   │        ↓
   │     返回首页
   │
   └─→ AI对话（7页）
            ↓
         随时返回
```

## 关键交互细节

### 1. 加载状态

| 场景 | 展示 |
|------|------|
| 拍照上传 | loading圈 + 「AI识别中...」 |
| AI回复 | 「小减正在输入...」 |
| 数据同步 | 顶部刷新动画 |

### 2. 空状态

| 页面 | 空状态文案 |
|------|------------|
| 首页无记录 | 「今天还没记录哦，从第一餐开始吧 🍽️」 |
| 体重无记录 | 「还没有体重记录，快来打卡吧 ⚖️」 |
| 统计无数据 | 「记录7天后，就能看到趋势图啦 📊」 |

### 3. 错误处理

| 错误 | 提示 |
|------|------|
| 拍照识别失败 | 「AI没看清楚，再拍一次试试？」 |
| 网络错误 | 「网络开小差了，稍后再试」 |
| 服务器错误 | 「服务器累了，休息下再来」 |

---

## 设计原则总结

1. **3步完成记录** - 拍照→确认→添加
2. **数据实时可见** - 圆环图随时展示进度
3. **AI无处不在** - 每个关键节点都有鼓励
4. **不打扰用户** - 鼓励简短，不弹窗打断
5. **正向激励** - 即使吃多了也不批评，给建议
